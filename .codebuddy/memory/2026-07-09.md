## Fixed GPU rendering (vello + wgpu, `--features gpu` / `--gpu` flag)
GPU path was panicking in `src/renderer/gpu.rs`. Three fixes applied:
1. Device limits: `wgpu::Limits::downlevel_defaults()` caps `max_storage_buffers_per_shader_stage` at 4, but vello 0.7 needs 5 → now use `adapter.limits().clone()`.
2. Target texture needed `TextureUsages::STORAGE_BINDING` (vello `render_to_texture` requirement) in addition to `RENDER_ATTACHMENT | COPY_SRC`.
3. Readback `bytes_per_row` must be 256-byte aligned (`COPY_BYTES_PER_ROW_ALIGNMENT`); pad to aligned stride and de-pad rows when copying into the tight RGBA buffer.
- Verified: `candy build examples/dot_move.tyx --gpu --format mp4` now succeeds (GPU adapter found = llvmpipe software rasterizer in this headless env).
